# Security Policy

## Supported Versions

This project does not have formal releases, as it primarily consists of documentation and modified app files for educational purposes. Security updates, if any, will be applied to the main branch.

| Version | Supported          |
| ------- | ------------------ |
| main branch (Threads app v416.0.0.29.66) | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly. We appreciate your efforts to help keep the project secure.

- **Where to report**: Open a private issue on GitHub by marking it as a vulnerability report, or contact the maintainer directly via GitHub (MUH4MM4D-SH4KIB) or email if available (check the maintainer's profile for contact details).
- **Response time**: You can expect an initial acknowledgment within 48 hours. We will investigate and provide updates on the status every 7 days until resolved.
- **What to expect**:
  - If the vulnerability is accepted, we will work on a fix and credit you (if desired) in the update notes.
  - If declined (e.g., not reproducible or out of scope), we will explain the reasoning.
- **Disclosure**: Please do not disclose the vulnerability publicly until it has been addressed and a fix is available. We follow responsible disclosure practices.
